# AIアニメスタジオ ブランディング戦略リサーチ — Claude Code 指示書

## 指示まとめ
- **目的**: 既存アニメ・アニメーションスタジオの成長事例を調査し、当社のポジショニング決定の参考にする。
- **成果物**: (1) 十字ポジショニングマップ（横軸＝受託メイン⇔オリジナル制作メイン、縦軸＝人数規模、設立時・現在の両方でプロット）、(2) リサーチレポート（事例一覧・根拠データ・出典一覧）。
- **軸**: 横軸＝受託/オリジナル（売上・作品数・IP保有主体で判定）、縦軸＝人数規模（設立時と現在を別記載）。
- **Phase 1**: タスク1（国内）とタスク2（海外）を並列でリサーチ。各スタジオについて「スタジオ名・国・設立年・設立時規模・現在規模・受託/オリジナル傾向・根拠・代表作品」を収集。タスク3で設立時/現在の2時点を明示して整理。
- **Phase 2**: 収集データでスケールを決め、十字マップに設立時・現在をプロット。凡例・出典を明記。
- **Phase 3**: マップ＋根拠データ一覧をレポートにまとめ、空白地帯・競合密集のメモを追記。データソースはレポート末尾に一覧記載。
- **出力**: 共通フォーマット（YAML/表/JSON）で保存し、マージ・マッピングに使う。完了後に報告レポートを提出。

## 目的
既存アニメ・アニメーションスタジオの成長事例を調査し、当社事業のポジショニング決定の参考資料とする。

## 成果物
1. **ポジショニングマップ（十字）**
   - 横軸: 受託メイン ⇔ オリジナル制作メイン
   - 縦軸: スタジオ人数規模（小 ⇔ 大）
   - 各スタジオを「設立時」と「現在」の両方でプロット
2. **リサーチレポート**
   - 国内外のアニメーション・アニメスタジオ事例の一覧と要約
   - マッピングに用いた根拠データ（設立年、規模推移、受託/オリジナル比率など）

## 軸の定義（データ収集時に統一する）
| 軸 | 左/下 | 右/上 | 判定の目安 |
|----|--------|--------|-------------|
| 横軸 | 受託メイン | オリジナル制作メイン | 売上・作品本数・代表的なIPの保有主体 |
| 縦軸 | 人数少 | 人数多 | 正社員・常駐スタッフ規模（設立時と現在を別記載） |

## 実行タスク

### Phase 1: リサーチ（並列推奨）
- **タスク1**: 国内アニメスタジオの事例収集（設立年、規模推移、受託/オリジナル、代表作品）
- **タスク2**: 海外アニメーションスタジオの事例収集（同様の項目）
- **タスク3**: 各事例について「設立時」と「現在」の2時点のデータを明示して整理

### Phase 2: マッピング
- 収集データから横軸・縦軸のスケールを決定
- 各スタジオを「設立時」「現在」でプロットし、十字のポジショニングマップを作成
- マップの凡例・出典を明記

### Phase 3: レポート
- マップの画像/図と、根拠データ一覧をレポートにまとめる
- 当社ポジション検討用のメモ（空白地帯・競合密集地帯など）を追記

## エージェント向け指示（並列リサーチ用）
- エージェントチームを組み、**タスク1** と **タスク2** を並列実行する。
- 各エージェントは「軸の定義」に従い、以下の項目を欠けずに収集する:
  - スタジオ名、国/地域、設立年
  - 設立時の人数規模（概数で可）
  - 現在の人数規模（概数で可）
  - 受託/オリジナルの傾向（受託メイン / オリジナルメイン / 混合）とその根拠
  - 代表作品・IP 1〜3件
- 収集結果は共通フォーマット（例: 表またはJSON）で出力し、Phase 2でマージしてマッピングに使用する。
- 完了後、報告レポートを提出する。

## 出力フォーマット（データ）
リサーチ結果は次の形式で保存すること（マージ・マッピングのため）:

```yaml
# 例: studios.yaml または同様の構造
studios:
  - name: "スタジオ名"
    country: "JP" # JP / US / FR など
    founded: 20XX
    size_at_foundation: "少/中/大 または 人数"
    size_current: "少/中/大 または 人数"
    axis_commission_vs_original: "受託メイン" | "オリジナルメイン" | "混合"
    evidence: "根拠の短い説明"
    notable_works: ["作品1", "作品2"]
```
